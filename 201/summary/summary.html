Inline elements

Inline elements occupy only the space of the content. They do not take width and height values.

Example:

Anchor tags are normally inline elements. You see they only take up the width of their content and follow each other on the same line.

Home Portfolio Contact

What happens if we set a margin property on an inline level element?

Inline elements will only take left and right margin values.

margin: 30px; margin: 30px; margin: 30px;

In the example I put a margin of 30px on all sides. As you can see if you check the anchor tag in the developer tools: only the left and right side of the contents are affected by the margin.

What about padding?

Inline elements accept padding values on all sides. But the top and bottom padding will spill into surrounding elements.

pading: 30px; pading: 30px; pading: 30px;

Floating inline elements

Floating inline elements will change its display value to a block level element and thus are subject to all block level characteristics. Which means they can take margin and padding values, as well as height and width values.

float: left; float: left; float: left;

Block elements

Block level elements take up the entire width of the page and always start on a new line. They can have margins and paddings. If no width is set, they will expand naturally to fill their parent container. If no height is set, they will expand to fit their content.

Example

(Note that div elements are naturally block level elements.)

This is a div with no width and height set.
This a div with no width but with a height of 60px, with a padding and margin of 20px around it.
This a div with a width of 90px and no height set.

In the first example, the div fills the entire width, and uses just enough height to fill its content. In the second box, we didn't set a width, but it still filled the entire width (counting the padding and margin); it also uses 60px of height plus the margin and padding. For the third box, we set the width to 90px, and as expected, it took just enough height to fill its content. The text was squished because of the reduced width, so the browser expanded the height to accommodate the content.

Floating block elements

Floating block elements with no width and height set are shrunk to the widest element within each div, in the following example the texts inside.

float: left;
float: right;

Since div elements are block elements, we can specify width and height, as in the following example.

float: left;
float: right;

Can we change an inline element into a block element?

Yes, we can change display values from a block element to inline element and vice versa by overriding the display value. In the following example, anchors are changed to block elements.

display: block; display: block; display: block;

Inline-block elements

Inline-block elements are inline elements that can take block elements properties, such as width and height. They act similarly to floated elements.

Example

display: inline-block; display: inline-block; display: inline-block;
